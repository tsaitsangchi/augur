用戶（2026-06-11）授權 augur 開發採「**有界自主**」模式（`/loop` self-pace），不再逐步等指令；理由：在「如何執行」這層 AI 比逐步等指令更能高效推進。**已正式入憲 `CLAUDE.md #26`（v1.2）**，含三條：① 自我糾錯主動（執行層改正確/補完整不被動問）② **以真兆為據、凡解釋／判斷／回答／做法皆先以 probe/實測/API/code 事實驗證、不靠「我以為」+ 主動自我追問「有沒有更近事實的做法」**（2026-06-12 強化；教訓：靠推測解釋 `FULL_START` 一路越描越錯、被 code 事實逼正）③ 碰護欄停。另 **#27 試錯即進步**（可控風險逐級逼近最佳奇異點、重覆驗證再定論）。目標只引用靈魂 SSOT、不複述（守 #12）。**2026-06-12 升級入治權層**：用戶要求「AI 自己 prompt 自己、執行層比用戶更清楚怎麼做、凡決策實證不憑『我以為』」→ 新增 **原則精華 #20 自驅動×實證決策**（憲章/靈魂/CLAUDE 同步；治權檔升 原則精華 v1.6.0 / 靈魂 v1.2.0 / CLAUDE v1.3）——「自驅動主動 × 實證決策」一體兩面（主動給速度、實證給正確、**無實證的主動必闖禍**）；CLAUDE #26 強化「自己 prompt 自己 (loop) ＋ 執行層方法 AI 主導」。實證動因：憑臆測的進程 pattern 連兩次誤殺（先殺自己 shell、再殺借用 venv 的 IDE python）。

**護欄內高度自主**（自主推進、不必逐步確認）：F3 models/evaluation 開發、寫測試、v1.4.0 code 落地、研究、規劃——凡 clean-room、可逆、不需外部副作用者。⚠**2026-07-17:「F3 開發卡 API 護欄」為 2026-06 史況(F3 早已完成、專案已進 arena live 期);紀律本體(#26 三條/#27)仍現行不變**。

**碰護欄即停、回報問用戶**（augur 原則硬性要求，非保守）：
- **API 放量抓取**（#24/#25：IP-ban 是 augur 最痛教訓，須最小探測先行 + 有人看顧）
- **commit / push**（#14：須明示授權，授權不延伸下次）
- **改 doctrine / 治權檔**（系統法律，AI 不自主改）
- **破壞性操作**（#6：不確定停手）
- **真兆最終確認**（#15：資料/結論「真兆還假兆」需用戶北極星判斷——三敵人「以為懂其實自欺」的防線）

**Why**：augur 靈魂「**系統建議、人決策；有紀律的顧問，不是自動駕駛**」（核心思想 line 114）——精神同樣適用於開發 augur 本身。

**How to apply**：每輪「評估狀態 → 選護欄內最優先 → 執行驗證 → 回報」；碰護欄即停。**已知瓶頸**：F3 需 feature store + universe 就緒 → 需完整 raw → 需 API 放量（卡 API 護欄）；故純自主先推「code 開發 + 不打 API 的單元測試」，資料管線/整合實測需用戶在 API 護欄授權。連結 [[augur_project_overview]]。
