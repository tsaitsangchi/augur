## 3. 三敵誠實邊界 — 為何「最佳／最精準」是假兆

**(1) 違靈魂成功定義。** 靈魂明訂「成功＝經濟價值（淨 Sharpe/Calmar/MaxDD）**非 IC**」「經濟價值≠準確率」。追「最精準」把目標偷換成 IC/準確率；而 augur 自己實證 **IC 撐住 ≠ 可交易**（B 關 H60 IC +0.152 很強，但只在 long 尾轉成經濟價值，long-short 近期崩、H20 經濟上已死）。優化「最精準」會誘導對 in-sample IC 過擬合 → 資料窺探 → 回測很美、實戰崩盤，正是敵人②③的生成路徑。

**(2) 追精準如何誘發自欺。** 「重新創造最精準模型」＝**無界模型搜尋**。在 n=8~25 非重疊期的小樣本上，每多試一個架構/超參都是一次多重比較。而按 SOP，試驗數 N 必須由 `trial_ledger` 機械計數去 deflate headline——**這個閘還沒建**（實查無此表）。等於現在「廣撒網試更多模型」是在**沒有懲罰機制的情況下盲搜**，每次都在抬升整個專案 headline 的樂觀偏誤而無從扣回。這是 #15「自我欺騙」的教科書入口。

**(3) 市場現實使「最佳」無法保證。** alpha 是市場效率的稀缺殘差，會被套利、會 regime 衰減（STAGE D 已見 long-short 是 2014-2020 regime 產物、2021 失效）。「2026 最佳／最精準」是無 out-of-sample 保證的絕對宣稱——真兆只存在於「這個 config、這個樣本期、out-of-sample 撐住、扣成本後勝基準、且揭露 caveat」的具體受限陳述。augur 現有成果正是後者（H60 LO 淨 Sharpe ~1.2、alpha 僅 long 側、n 小），它**誠實地拒絕「最佳」字眼**。

**(4) Claude 的真實角色。** Claude 能做的是**嚴謹的紀律執行者**——反洩漏把關（#8）、誠實驗證（#15）、多 seed + HAC-t（#11）、經濟終關（#14）、預先登記防窺探。Claude **不能變出不存在的 alpha**。「報酬非試錯對象」：可試的是方法/參數/horizon/宇宙，**絕不試「把精準度目標當放鬆資料誠實的藉口」**。移植任何外部「最佳模型」碼違 clean-room #16-17（「任何移植痕跡＝治權違規」）——這不是效率取捨，是身分定義。

## 5. 建議優先序 + 分階段（每階段：元件 / 驗收＝#14 + 反洩漏；撐住才升，不過誠實淘汰）

### 階段 0 — 釘實地板（前置，不做則後面全部無效）

- **升/淘汰**：deflated net Sharpe 仍 ≥1.0 → 升；明顯崩 → 誠實記「headline 本為樂觀」，重估後續價值。

### 階段 1 — 消 survivorship 債 + 成本 realism（把條件性 PASS 變真 PASS）
- **元件**：(C) #25 最小單位探 FinMind 下市日可得性 → 方案 A point-in-time roster；(D) 成本模型含滑價/衝擊/借券。

### 階段 3 — 模型側唯一實驗（只有走到這才動模型）
- **元件**：(E) LambdaRank 損失對齊當挑戰者；(F) 等權 ensemble。
- **驗收（#14 全套）**：≥3 seed 正增量 + HAC-t≥2 + **deflated 淨 Sharpe 須真贏現 Ridge**（且 N 已含這些新試驗）；**IC 升不算數，經濟終關才算**。
- **升/淘汰**：真贏才提拔且**須用戶拍板**（系統建議、人決策，不自行切換部署主模型）；預期是邊際增益/穩健性、非躍升——過不了就誠實記「無增量」（負結果也有價值）。

## 6. 誠實 ceiling — 不承諾魔法精準

- **本機 CPU + FinMind 資料量 → DL 不可行**：不是算力問題，是**樣本問題**。有效期數 n=8（H120 近期）~25（H60 全期），文獻示須 N≥750-1500 才收斂——深度模型過擬合幾乎必然。（註：此樣本量閾值為外部 web 一般性方向支持、非本次實查 augur 數字，僅作定性參考。）
- **edge 本質是稀薄、易衰減**：現有 Ridge H60 LO 的優勢是歷史 walk-forward 結果，不保證未來；long-short 已示 regime 衰減。正解是「持續再驗證 + 誠實判停」，不是追更精準。
- **所有排名是方向性非精確**：`model_registry.note` 自標「排名方向性非精確保證」；n=8~18 下兩模型 Sharpe 差 0.1~0.3 可能在抽樣誤差內。任何「補了 X 後 Sharpe 升到 Y」的宣稱**須作者親跑同口徑自證、不轉述 agent**（memory 教訓）。
- **survivorship 債未閉環＝現存已知樂觀偏誤**（`model_registry.note` 白紙黑字）：所有 as-of IC/經濟數字帶未量化樂觀偏誤，這是所有模型共同天花板，**換更強模型反而更容易擬合這個偏誤**。
- **本報告未親跑任何新模型/新特徵實驗**：upside/風險是基於實查 ledger + 報告 + 方法論的方向判讀，非本次新跑數字。任何模型變更定論須實際過 STAGE B→C→D 三閘親跑自證。

## 7. 不做什麼（明文）

1. **不追「最佳模型／最精準模型」神話**——「最佳/最精準/保證」是治權要求先當假兆檢驗的紅旗詞；不存在可移植的「2026 最佳選股模型」。
2. **不移植任何外部模型 code/數字/設定**（clean-room #16-17）——即使某外部「最佳模型」真有效，移植其碼＝治權違規。augur 只能從 5 治權檔 + 自身 schema + live API 實證長出。
3. **不在 n=8~25 小樣本上做 NN/Transformer/AutoML 超參搜尋**——過擬合三要件全備，且每 trial 抬 deflation N（而 deflation 閘還沒建，等於盲搜）。
4. **在 deflation 閘建成前，不引用 1.20 作為終判、不對用戶宣稱「已驗證可交易」**——必附「12 選 1、deflation 之前、單 seed、僅比例成本」標籤。
5. **不為報酬鬆動三敵**——可試方法/參數/horizon/宇宙/損失函數；**絕不試資料真假**（假資料/偷看未來/自我欺騙永遠零容忍）。追精準若犧牲誠實，連「不是假兆」都保不住。
6. **不自行切換部署主模型**——模型提拔屬決策層，須過完整 #14 + **用戶拍板**（系統建議、人決策）。
