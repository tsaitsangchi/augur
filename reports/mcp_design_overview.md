# MCP Design Overview – Token Optimisation for Augur

## 🎯 Goal
Create a **Multi‑Channel Proxy (MCP)** layer between Augur tooling (e.g. `constitution_lint`, report generation) and Claude Desktop so that:
- Repeated prompts are **cached** → no extra token cost.
- Cheap, deterministic queries are answered by a **local LLM** (or static stub) instead of Claude.
- High‑risk compliance checks continue to use Claude Desktop with **audit logging**.
- All traffic passes through a **router** that automatically selects the appropriate backend.

---

## 📦 Architecture Diagram
```
+-----------------+       +-------------------+       +-------------------+
| Augur tooling   | ----> |   MCP Router      | ---> | Claude Desktop   |
| (python scripts)|       | (FastAPI)         |       | (CLI)            |
+-----------------+       +-------------------+       +-------------------+
                               |
                               |   +-------------------+
                               +---| Cache (Redis)    |
                               |   +-------------------+
                               |
                               |   +-------------------+
                               +---| Local LLM Wrapper |
                                   (e.g. Ollama/Mistral) |
                                   +-------------------+
```
- **Router** (`mcp/router.py`): receives JSON `{prompt, type}` and decides which backend to call.
- **Cache** (`mcp/cache.py`): `hash(prompt+type) → response` with TTL per type (24 h for compliance, 7 d for static docs).
- **Classifier** (`mcp/classifier.py`): rule‑based (keywords, regex) to label a request as `quick`, `compliance`, or `audit`.
- **Local LLM** (`mcp/local_llm.py`): runs a tiny open‑source model (Mistral‑7B‑Instruct) via Ollama; for the MVP we ship a stub that returns a canned answer.
- **Claude Wrapper** (`mcp/claude_cli.py`): executes the installed `claude` CLI, parses the token usage line, and returns `{response, tokens}`.
- **Audit Logger** (`mcp/logger.py`): writes JSON lines to `logs/mcp_audit.log` (timestamp, user, backend, token count, prompt hash).

## ⚙️ Runtime Flow (example)
1. **Augur script** wants a compliance summary → calls `invoke_mcp(prompt, type="compliance")`.
2. **Router** receives request, asks **Classifier** → decides `backend = "claude"`.
3. **Cache** is checked; miss → Router forwards to **Claude Wrapper**.
4. **Claude Wrapper** runs `claude -p "$PROMPT"` and captures output + token count.
5. Response is stored in **Cache** and returned to the script.
6. **Logger** writes a line:
   ```json
   {"ts":"2026-07-21T10:00:00Z","type":"compliance","prompt_hash":"a1b2c3","backend":"claude","tokens":84}
   ```
7. Subsequent identical prompt hits the cache → **0 tokens** consumed.

## 📊 Token Optimisation Benefits (estimated)
| Scenario | Avg. Tokens / Call | Calls / CI Run | Tokens Saved |
|----------|-------------------|----------------|--------------|
| Compliance check (full report) | 120 | 1 | 0 (still Claude) |
| Small summary (e.g., L0‑L1) | 30 | 5 | ≈120 (cached after first) |
| Quick factual query (e.g., "Explain L2") | 25 | 10 | ≈250 (served by local LLM) |

With caching, the **first run** still incurs token cost, but subsequent runs (common in CI loops) become free. Over a month of nightly CI, we expect **≈2 k tokens** saved – roughly **$0.20** at current pricing.

## 📋 Monitoring & Alerts
- **Grafana dashboard** reads `logs/mcp_audit.log` (via Loki) – shows daily token consumption, cache‑hit ratio, and backend distribution.
- **Alert** when daily token usage spikes > 10 % of baseline → Slack/Email notification.
- **Health endpoint** `GET /health` returns `{"status":"ok"}`; CI waits for `200` before proceeding.

## 🛡️ Risks & Mitigations
| Risk | Impact | Mitigation |
|------|--------|------------|
| stale cache → outdated compliance data | False‑positive/negative lint results | TTL per type (24 h) + CI flush (`redis-cli FLUSHALL`) before each run |
| local LLM giving inaccurate answers | Mis‑guidance in low‑risk queries | Use it only for `type="quick"` (non‑compliance) and clearly mark answers with `(local answer)` |
| network/port conflict on CI agents | Build failure | Choose high, unused port (e.g., 8123) and expose via env var `MCP_PORT` |
| token‑logging privacy leak | Sensitive prompt exposure | Hash prompts (SHA‑256) before logging; store only hash, not full text |
