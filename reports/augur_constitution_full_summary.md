# Augur 憲章完整概覽（L0 → L7）

## 🌈 為什麼需要「元憲章」
- **防止失控**：任何 AI 行動都必須先取得**可驗證的人類授權**，避免自動化系統自行決策、產生不可預測的影響。
- **透明可審計**：每一步都有 **來源、時間、簽章**，全鏈可回溯，任何異常都能即時追蹤。
- **人類治理**：最高決策權屬於真實的 **憲章管家（Constitution Steward）**，AI 只能執行、不能解釋或修改規則。

---

## 📜 L0 – 世界演化的官方流程（12 步）
```
真實世界 → 觀測 → 表徵 → 辨認身份 → 以證據引用 → 建立知識
      → 推理 → 規劃 → 【人類授權關卡】 → 行動 → 回饋 → 學習
```
| 步驟 | 簡述 |
|------|------|
| **觀測** | 從感測器、資料庫或外部 API 把真實世界的原始事實抓下來。 |
| **表徵** | 把觀測結果轉成 AI 能處理的符號（向量、文字、表格等）。 |
| **辨認身份** | 確認資訊屬於哪個實體或概念，避免混淆。 |
| **以證據引用** | 每筆資訊必須能追溯到可信來源（文件、專家、法規）。 |
| **建立知識** | 從證據抽象出概念、規則或模型。 |
| **推理** | 依據已建立的知識作出邏輯或統計推斷。 |
| **規劃** | 設計具體行動方案，列出步驟、資源與風險。 |
| **人類授權關卡** | **唯一阻斷點**：AI 必須取得 **書面、可驗證** 的人類授權才能繼續。 |
| **行動** | 執行方案，對真實世界產生實際影響。 |
| **回饋** | 再次觀測執行結果，產出新資料。 |
| **學習** | 用回饋更新模型或規則，形成閉環改進。 |

### 人類授權關卡核心要素
- **書面授權**：明確文字或簽章，記錄授權者（憲章管家）與時間。  
- **五道品質關卡**（G1‑G5，見下方 L0 質量保證）必須全部通過才能產生授權。  
- **不可逆**：行動完成後必留下完整觀測紀錄，供未來審計。

## ✅ L0 質量保證 – 五道關卡（G1‑G5）
| 關卡 | 核心任務 |
|------|-----------|
| **G1 首審** | 三面鏡子檢查：**一致性、嚴謹度、現實性**。 |
| **G2 處置** | 重大問題必由 **憲章管家** 正式裁決。 |
| **G3 窮舉** | 機械式逐條檢查，避免口頭「沒問題」的漏網。 |
| **G4 補列** | 針對 G3 發現的缺口全部補齊，無遺留。 |
| **G5 複驗** | **獨立真人**最終驗證，簽章確認。 |

## 📈 L1 – 資料層面的「真理」
- **可追溯**：每筆資料必記錄 **來源、時間、簽章**，形成不可篡改的鏈條。
- **分層結構**：
  - **原始觀測** – 未加工的感測資料。
  - **證據層** – 已驗證、可引用的事實。
  - **知識層** – 從證據抽象出的概念與規則。
  - **推理層** – 基於知識的演繹或歸納結果。
- **授權標記**：每筆可執行的「行動」必帶 **`<human‑authorized>`** 標記，系統執行前自動驗證其真偽。

---

## 📚 L2 – 規範語言（MD‑X）
- 使用 **Markdown‑X (MD‑X)** 作為正式語法，兼容普通 Markdown 同時加入 **機械可解的結構化標記**（如 `<!-- lint:KEY -->`）。
- 章節、條款、子條款都有唯一 **ID**（如 `WM.40`、`P1.E1`），方便自動 lint、搜尋與追溯。
- 每條款必宣告 **來源**（`source: MC` 或 `source: AUGUR‑XX`），缺失即視為 **未分類**，在統計時單獨列出。

---

## ⚙️ L3 – 執行框架（tools/constitution_lint）
- **selftest**：零外部依賴的紅綠自檢，保證所有規則、回饋、授權流程都能正確運作。
- **compliance_lint**：對規格檔（`specs/*.md`）做合規檢查，必須全部 **PASS** 才允許 CI 繼續。
- **audit_lint**：檢查程式碼是否符合憲章規則（如禁止未授權的寫入、禁止自動改寫憲章等）。
- **report**：產生機械可讀的 JSON 與人類可讀的 Markdown 報表，包含 **coverage、錯誤、警告** 統計。

---

## 🛡️ L4 – 治理模型（Constitution Steward）
- **角色**：真人憲章管家，負責 **解釋條文、審查違憲、裁決修憲**。
- **權限**：只能 **授權**、**解釋**、**修改** L0‑L7 的規則，AI 完全無權參與。
- **門檻**：修憲必須提供 **書面證據**，並證明新版更好；修訂後需重新通過所有五道關卡。

---

## 🔧 L5 – 技術基礎設施
- **Git repository**：所有憲章文件、工具程式與自檢報表都存於同一 repo，確保單一真相來源。
- **CI/CD**：GitHub Actions 觸發 `constitution‑lint.yml`，自動跑 `selftest`、`compliance_lint`、`audit_lint`；任一失敗即阻止 merge。
- **Artifacts**：每次 CI 產出的報表、統計會自動上傳為 workflow artifacts，方便查閱與審計。

---

## 📊 L6 – 統計與度量（report 模組）
- **coverage**：自檢會自動計算 **測試覆蓋率**（如 `✓ 47 項`），確保每條規則都有被驗證。
- **error / warning** 統計：分別列出 **machine‑generated** 與 **human‑generated** 的問題，便於快速定位。
- **同步標記**：`--sync` 參數會把報表數字寫回 `[I]` 標記的檔案，保證文件與程式的數據永遠一致。

---

## 🌐 L7 – 生態系統與未來願景
- **跨專案共用**：所有使用 Augur 憲章的子專案（如 `augur‑knowledge`、`augur‑ops`）只要引用同一套 **MD‑X** 規範，即可共享相同的治理與審核流程。
- **持續演化**：新規則必經 **L0‑L5** 全流程審核後才會發布，確保每次變更不會破壞既有安全防線。
- **開放透明**：所有規則、審核結果、CI 報表均公開於 GitHub，任何人可檢視、審計，增強信任。

---

## 📄 小結
- **L0‑L7** 構成了 **Augur 元憲章** 的全套防護機制：從 **「世界觀測 → 人類授權 → 行動 → 追溯」** 的完整鏈條，到 **治理、技術、統計** 的全方位支撐。
- 只要遵循 **五道品質關卡**、使用 **MD‑X**、在 **CI** 中跑 **selftest / compliance / audit**，任何 AI 或自動化系統都會被嚴格限制在 **「人類授權」** 的安全閘門之內。
- 本報告以 **純文字/Markdown** 呈現，適合非技術讀者快速了解 Augur 憲章的精神與運作流程。
