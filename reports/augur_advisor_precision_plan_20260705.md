# augur 顧問精準度改善計畫（回答很像 + 引文不相關）

🎯 **白話**：顧問「誠實博學的我」目前**不管問什麼都套「多視角/逆向鏡/週期觀」三姿態、內容雷同**，且**引文常不相關**（問會計卻引論語/稅賦/斯賓格勒）。本計畫依 ultracode 四視角逐行核實 code 找根因、設計改善——**讓定義題直接答準、投資題保留縱深、離題走誠實句，且誠實鏈（advise+guard 單閘）與三敵零容忍完全不鬆動**。依 plan-first：**先計畫、拍板後實作**。
守 #1（引文逐字不編）· #15（答不出誠實說不知道、不硬掰）· 憲章 v1.30.0（顧問呈現層）· v1.33.0（執行前確認）· 誠實鏈不可繞。日期：2026-07-05。四視角診斷（prompt/檢索/誠實張力/模型設定）逐行核實 code。

## 0. 問題與目標

### 0.1 兩症狀（用戶實測）
| 症狀 | 表現 | 例 |
|---|---|---|
| **A. 回答很像** | 不論問什麼，答案硬套「多視角→逆向鏡→週期觀」三段 | 「會計是什麼」被逼出空洞三段 |
| **B. 引文不相關** | 答案飄 `[1][3][5]` 指向與題無關的經典 | 問會計，引論語（君子/稅賦）、斯賓格勒（文明衰落） |

### 0.2 兩症狀同源互放大（不是兩件事）
```
檢索無門檻(kNN 必回 top-k)   →  citations 恆非空(#15 誠實出口幾乎打不到)
        +                                    ↓
prompt 硬套三姿態(每段要標 [N])  →  模型為填滿三段、每段掛一個 [N]
        +                                    ↓
弱模型 qwen3:8b + 低溫 0.15       →  最省力解＝把不相關 top-k 硬塞進三段模板
```
「必須三段」製造「必須引」的壓力，「檢索無門檻」剛好供應低相關結果去填坑。**兩邊都修才根治。**

### 0.3 目標（守恆）
- 定義/概念題 → 直接給清楚定義，不硬套三姿態、不硬引古文。
- 投資分析題 → **保留三姿態縱深（顧問靈魂設定，不砍）**。
- 離題/庫外題 → 檢索濾空後走既有 `#15` 誠實句閉集（不硬掰）。
- **誠實鏈（advise+guard 單閘）零改動；三敵（假資料/偷看未來/自我欺騙）零容忍不因「答得更準」而鬆動。**

## 2. 改善設計（逐元件，附 code 改法 + 檔·函式）

### 2.1 元件 A — prompt 問題自適應框架（治 R1/R2/R3/R6）· 檔 `advisor/prompt.py`
**A-1 零 ML 確定性題型偵測**（仿 `retrieval._guess_language`，可單測）：
```python
_DEFINE_CUES=("是什麼","什麼是","定義","怎麼運作","如何計算","什麼意思","what is","define","how does","explain")
_INVEST_CUES=("該不該買","加碼","減碼","停損","停利","選股","標的","值得買","追高","看好","看空","進場","出場")
_TICKER=re.compile(r"\b\d{4}\b")   # 台股四位代號
def _query_kind(query):
    q=query or ""
    if any(c in q for c in _INVEST_CUES) or _TICKER.search(q): return "analysis"
    if any(c in q for c in _DEFINE_CUES): return "definition"
    return "general"
```
判準**保守**（#15）：不確定 → `general`（維持現行混合），寧可保留三姿態也不誤殺；誤判最壞＝風格瑕疵，**不污染資料、不觸誠實鏈**、可恢復試錯（#27）。

**A-2 SYSTEM_PROMPT 格式段改條件式**（L17-30）：
- L18 → 「先判斷問題類型。純定義/概念題直接一兩段講清楚講準，**不硬套多視角、不為湊格式分三段**；投資分析/標的判斷題才用三姿態展開縱深。」
- L20 標題 → 「三種認知姿態（**僅投資分析題適用；定義題不必用**）」
- 補**引文相關性門檻句**（治 R6）：「`[N]` 只在該段引文**內容真的支持你這句話**時才標；檢索清單裡沒有與本題相關內容，就**不要引任何 [N]**，用自己的話回答（引文是輔助、不是必填）。」

**A-3 build_prompt 注入題型 hint + 改中性結尾**（L55-73）：入口 `kind=_query_kind(query)`；`definition` 時附「【本題為定義題】直接給清楚定義、不套三姿態、不硬引不相關古文」；L73「請依三姿態…」→「依上方原則作答：定義題直接講、分析題才三姿態；引文相關才標、不相關不硬引（寧可不引不可硬湊）；不打引號、不照抄古文。」
> **架構可行性已證**：`build_attached_prompt`（Mode B）本就「換框架、不套三姿態、guard 不變」在用 → 題型自適應零架構阻力。

### 2.2 元件 B — retrieval 相關度門檻 + 弱檢索走通識（治 R4/R5）· 檔 `philosophy/retrieval.py`
**B-1 語義 kNN 加相關度門檻**（`retrieve()` + `retrieve_items()` ANN 段）：簽名加 `min_score=None`（None=沿用舊行為，**門檻由呼叫端給、不 hardcode 於函式體** #27）：
```python
def retrieve(query,k=8,work_id=None,scope=None,min_score=None):
    ...
    if min_score is not None: out=[c for c in out if c.score>=min_score]
    return out
```
`retrieve_items` ANN 分支同法；**exact 命中（詞面重疊 via='exact'）不套 cosine 門檻**（score 口徑不同）。

**B-2 retrieve_all 改相關度驅動配額**（L312-332）：works 不再強佔固定半數，三路徑各過門檻後**以 score 全域排序取 top-k**（取代 `zip_longest` 固定交錯）：
```python
merged=sorted(works+pub+priv,key=lambda c:-c.score)[:k]
```
works 無夠相關 work 時自然縮到 0。**RBAC 不動**：works 公開/items 收窄（clean_item_sql）是安全軸，只改「配額憑相關度而非路徑」，不碰收窄邏輯。

**B-3 domain 感知（可選、決策層，§5）**：確定性關鍵詞粗分 query 領域（仿 `_guess_language`、零 LLM #28）：財經題調小 works、調大 items；保守（不確定維持混合）。**偏決策層**（碰「顧問優先讀哪些素材」）、列後續。

**B-4 弱檢索走通識**：門檻生效後 citations 可空/少——**完全空** → 維持 `advise.py` `guard_empty_retrieval` 三級誠實句（**不改閉集**）；**非空但少** → A-3 已把「必引」改「相關才引」，模型用通識白話答、**仍受三硬約束（不引號/不照抄/數字只轉述 payload）+ guard 事後把關**。

### 2.3 元件 C — advise/guard「相關才引」邊界（**不改 guard 判準**）· 檔 `advise.py`/`guard.py`
- **guard.py 完全不動**：四道機械閘（引文逐字/數字白名單/未來語/逆向）原樣。「相關才引」是叫模型**別亂引**（prompt 層）、非放鬆 guard；**只會減少引文、不會讓模型編引文**（少引的一樣過逐字閘）。
- **advise.py 主線不改簽名**：B-1 門檻讓 citations 可空 → 既有空檢索分支**自動激活**（現因 top-k 恆滿幾乎打不到）→ 把 `#15` 誠實出口從「結構上不可達」變「真正可達」＝**強化 #15**。
- **可選硬閘（越界編號、零誤判）**：解析 response 內 `[數字]`，超出 `len(citations)`（只 3 條卻標 [5]）→ 標 issue。**純增檢查、不放鬆三敵**。相關性軟閘（詞彙重疊）**延後**（誤判率須先實測，**絕不反向豁免逐字閘**）。

### 2.4 元件 D — model config（治 R7/R8）· 檔 `serve_advisor_openai.py`/`payload.py`
- **D-1 非選股題不注入示範選股（治 R7）**：一般問答改傳**空 `KnowledgePayload`**（`sql_numbers=∅`、`picks=()`），`_payload_block` 對空 payload 印「（本回合無預測數字）」而非塞示範選股。**守 #1**：空 payload 下數字白名單為空、模型自造數字被 guard ② 攔（**更嚴不更鬆**）。
- **D-2 溫度/thinking 逐級試（治 R8、operational #27）**：`temperature 0.15→0.4~0.6`（破模板複讀）、think 視延遲量測；**逐級調、以 §4 量測校準退場、不單次入憲**；溫度升後 guard ①②③④ 全程作用、fail 走誠實句 → **誠實鏈由 guard 兜底**；須確認 **guard pass 率不劣化**，劣化即退回。

## 3. 誠實守法（改動**不**放鬆三敵，逐項界定）
| 改動 | 對 #1/#15/guard 之效果 |
|---|---|
| A 題型自適應 | 純 prompt 文字/框架，不觸檢索與 guard；誤判＝風格瑕疵、零資料污染 |
| B 相關度門檻 | 讓 #15 誠實出口**真正可達**（強化）；少引的引文仍過逐字閘 |
| B 弱檢索走通識 | 空 → 誠實句閉集（不改）；少 → 通識答仍受三硬約束 + guard |
| C guard | **一字不改**；可選越界 [N] 檢查＝純增防護 |
| D 空 payload | 數字白名單為空＝**更嚴**（自造數字必被攔） |
| D 升溫 | guard 四閘兜底、fail 走誠實句；以 pass 率不劣化為退場閘 |
**紅線**：相關性軟閘不得反向豁免逐字閘；三敵不是試錯對象（#27）；閉集/guard 判準變更＝憲章判準變更（不在本計畫）。

## 4. 精準度量測（不靠「感覺變好」、可驗證）
**測試問句集**（≥15 題，三類）：定義題（會計是什麼/PER 怎麼算/什麼是安全邊際）· 投資分析題（財報能當依據嗎/某標的該不該買）· 離題/庫外題（今天天氣/量子力學）。
**評分準則**（每題 0-2）：(a) **答得準**（定義題有無給對定義）· (b) **引文相關**（[N] 是否真支持論點、有無硬塞離題引文）· (c) **非千篇一律**（定義題有無被硬套三姿態）· (d) **誠實**（離題題有無走誠實句、有無硬掰）。
**機械指標**：guard pass 率（改善前後**不得劣化**）· 平均引文數（應下降於定義/離題題）· 三姿態出現率（定義題應大降）。**改善前先跑一次 baseline、改後對照**（#15 實證非自我說服）。

## 5. 分階段（優先序、風險）
- **Phase 1（high、低風險高影響、無需校準）**：A 題型自適應 prompt + D-1 非選股題空 payload。純 prompt/payload 文字，不觸檢索/guard，立刻改善「很像」+ 去除示範選股雜訊。**先做、先量測**。
- **Phase 2（high、需閾值校準）**：B-1 相關度門檻 + B-2 score 配額。先跑 score 分佈量測定閾值（e5-small 前綴分佈偏高，候選 0.35~0.82 依實測、標「實驗中」不入憲 #27），治「引文不相關」。
- **Phase 3（medium、決策層/operational）**：B-3 domain 感知（碰素材優先序＝偏決策層拍板）· D-2 溫度/thinking 試錯（#27 逐級、量測退場）· C 越界 [N] 硬閘（純增防護）。

## 6. 驗收準則
- **視覺**：定義題（會計是什麼）給直接清楚定義、不再三姿態；投資題仍有三姿態縱深；離題題走誠實句。
- **引文**：定義/離題題引文數大降、無離題 [N]；投資題引文與論點相關。
- **誠實**：guard pass 率不劣化、`#15` 誠實句在濾空時正確觸發、無自造數字外洩（空 payload 更嚴）。
- **量測**：§4 測試集改善前後對照有明確進步（(a)-(d) 分數升、機械指標達標）。
- **回歸**：`pytest` 全綠（含 `_query_kind` 單測）、隔離 AST 稽核過、guard 測試不變。

## 8. 實作與自問自答迭代結果（2026-07-05，有界自主 §26）

**已落地（Phase 1，執行層、未碰 guard/誠實閉集）**：A 題型自適應 prompt（`prompt._query_kind` + SYSTEM_PROMPT 條件式三姿態 + 引文/散文相關性門檻）· D-1 非選股題傳 `payload.empty_payload()`（去示範選股雜訊、數字白名單更嚴）。3 輪自問自答（實跑 qwen3、逐輪重啟 advisor）逐步收斂：

| 迭代 | 改動 | 自問自答結果 |
|---|---|---|
| **Iter 1** | A 題型自適應 + D-1 空 payload | 「會計是什麼」→ 直接準確定義、**0 三姿態/0 [N]/0 引經據典**;殘留末尾多「知識庫中無此內容」 |
| **Iter 2** | prompt (c) 修:定義/常識題可用知識直答、不畫蛇添足說「無此內容」 | 定義題**殘留消失**(含 IFRS/GAAP 完整定義);概念題去三姿態模板 |
| **Iter 3** | prompt 補:與題無關檢索內容**連散文借用都禁**(不硬扯哲學類比) | 概念題**斯賓格勒牽強類比消失**、視角扣財務實質;真決策題「2330 該不該買」**三姿態縱深保留**、逆向不翻轉(0 該賣/放空)、guard 過 |
| **Iter 4** | 補「特殊題型處置」段(未來預測/穩賺/離題創作/投資術語);繞 guard 觸發詞給有用誠實答 | 「安全邊際」→ **投資語境定義**(格雷厄姆/內在價值折扣、去材料強度雜訊)✅;「台積電明天會漲嗎」→ **有用誠實答**(短期不可預測、系統做機率排序非單日預測、避開明天漲跌觸發詞、guard 過)✅ |
