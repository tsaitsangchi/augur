# 系統架構大憲章 v1.2.0

**性質**：Augur 的最高治權文件——回答「系統**怎麼建、怎麼運作、怎麼治理**」。
**位階**：精神以 `docs/系統核心思想_v1.0.0.md`（靈魂）為準；不可違反之法律全文以 `docs/原則精華_v1.2.0.md`（17 條）為準（SSOT）。本憲章承載**架構分層 + 維運序列 + 升版規則**，並把那 17 條法律**對映到「三個敵人 × 管線」**框架上。
**脊椎**：整部憲章用兩條軸組織——
- **三個敵人（WHY）**：假資料 / 偷看未來 / 自我欺騙——系統存在的每條紀律都在防它們。
- **管線（HOW）**：`raw → feature → universe → model → validate` + 橫切（core / audit）。
**讀法**：第一部懂精神，第二部懂為什麼，第三部懂架構，第四部查法律，第五部照著重建，第六部知道怎麼改憲法。目標：**30 分鐘讀懂全系統法律**。

---

## 第一部　系統本質

Augur 是一套**只用真實資料、誠實預測台股**的系統：從 FinMind / FRED 抓進可溯源的日級資料 → 算成 source-pure 特徵 → 選出資料乾淨的核心股 → 訓練模型預測未來 H 日的**相對強弱**（橫斷面排序 → top-N）→ 以 walk-forward 誠實驗證並附**可信度**。

> 全文見 `系統核心思想_v1.0.0.md`。一句話：**只信真兆。**

| 面向 | 內容 |
|---|---|
| 標的 | 台灣上市櫃個股 |
| 任務 | 橫斷面相對強弱排序（週/月/季/年多尺度）→ top-N |
| 資料 | FinMind API + FRED；**日為最小單位** |
| 產物 | 分數/機率 + 可信度（walk-forward IC / Eff-t / 勝率） |
| 邊界 | 不日內、不保證獲利、不擇時躲崩盤、不用未來資訊（anti-leakage）、不下單動錢、不存無源值 |

## 第二部　三個敵人（防線總綱）— 為什麼有這些法律

量化系統最容易死在三件事上。17 條法律**全部**是在防這三個敵人（#17 為取得穩健之 precondition：封鎖則無真資料→一切下游停擺）。

### 敵人 ①　假資料（AI 幻像）
- **長相**：imputed / zero-fill / hardcoded 知識 / 推估 / 系統自己補的值——沒有真實 API 來源。
- **致命後果**：用假資料跑出來的模型沒有意義；錯誤從源頭污染一切下游。
- **防線（原則）**：**#1 零幻像 ★** · #2 API 即權威 · #3 純通用 ingestion · #4 日為最小單位 · #5 型別紀律 · #6 冪等+resume · #7 DB↔API 對帳 · #9 思想≠特定值（hardcoded 值＝注入假資料）· #10 核心股質>量（不完整股排除）· #16 clean-room（防 stock_backend 硬編值經 code 回流）· #17 API 速率公民（主動限速,防 IP 封鎖致無真資料）。
- **最危險的管線層**：`raw`、`feature`、`universe`。

### 敵人 ②　偷看未來（時間洩漏）
- **長相**：用了決策當下還不知道的資訊（未來函數 / 標籤窗洩漏 / 發布日錯置）。
- **致命後果**：回測很美、實戰崩盤——最隱蔽、最致命的自欺。
- **防線（原則）**：**#8 anti-leakage ★**（≤t 設定、t+1 套用、walk-forward IC 附 purged 口徑；含 survivorship/選股偏差防護）。
- **最危險的管線層**：`feature`（特徵時點）、`model`（訓練/驗證切分）、`validate`。

### 敵人 ③　自我欺騙（灌水回報）
- **長相**：把「打算做的」當成果、拿單次極值當定論、單看 gain 判特徵、以準確率宣稱有效、報喜不報憂。
- **致命後果**：文件失真比沒文件更危險——驅動錯誤決策。
- **防線（原則）**：#10 核心股質>量（不誇大覆蓋）· #11 五鏡特徵治理 · #12 單一引用源 · #13 空頭防護規則地板優先 · #14 經濟價值判定 · **#15 誠實回報 ★** · #16 clean-room（防未重建 code 之自欺回流）。
- **最危險的管線層**：`feature`（特徵存廢）、`validate`（成績宣稱）、報告層。

> ★ = 各敵人的**基石防線**（#1 / #8 / #15，即原則精華「三條基石」）——一條守一個敵人；守住這三條，等於守住三個敵人的核心。

## 第三部　管線分層架構 — 系統怎麼建

資料流：`raw → feature → universe → model → validate`，外加兩個橫切層 `core` / `audit`。
**鐵律：各層職責不越界**——選股不算特徵、特徵不抓 API、驗證不讀訓練產物。

### 橫切 · core（`src/augur/core/`）
- **職責**：路徑/設定、DB 連線、**generic_schema（auto-schema SSOT）**、schema 推導 helper（查 DB information_schema）、infra log 表 DDL（PHASE 1 bootstrap）。
- **守**：① — #2 API 即權威、#3 純通用、#5 型別紀律。
- **邊界**：不抓 API、不選股、不算特徵；只提供地基設施。
- **SSOT**：型別規則（字串 VARCHAR(255)/TEXT、數字 NUMERIC(20,6) 自動擴大）唯一住在這裡。

### 1 · raw（`src/augur/ingestion/`）
- **職責**：通用 ingester 把**任意** FinMind/FRED dataset 自動建表落地（無白名單）。
- **輸入→輸出**：API 回應 → DB 原始表（每值 byte-equal API）。
- **守**：① — #1 #2 #3 #4 #6 #7。
- **邊界**：不算特徵、不選股；intraday 不入庫（只存日級衍生，#4）。

### 2 · feature（`src/augur/features/`）
- **職責**：只用 source-pure 原始值，算成特徵面板（panel）；自建 `feature_*` 表（`CREATE IF NOT EXISTS`）。
- **輸入→輸出**：raw 表 →（每股每面板）source-pure 特徵；**算不出真實值的不寫入（缺列，不存 fake / 不存 zero-fill）**。完整度（哪些股缺哪些特徵）交由 universe 層判定。
- **守**：① 不造假特徵、**不存無源值**（#1 #9，扣靈魂「不存沒有 API 來源的值」）；② 特徵時點 anti-leakage（#8）。
- **邊界**：不抓 API、不選股；公式禁 hardcoded 知識/閾值（#9）。

### 3 · universe（`src/augur/universe/`）
- **職責**：核心股選拔——核心股 ＝ **全部「source-pure 完整」的股**（無評分、無排名上限）；任一面板任一特徵缺值（不完整）即排除；自建 `core_universe_*` 表。
- **輸入→輸出**：raw 表（完整候選）＋ feature 面板（完整度收緊）→ committed 核心股名單（**兩道完整度關，無評分**）。
- **守**：① source-pure gate（#1）、不完整股排除（#10）；③ 質>量（#10，可以少、不以數量為目標、**不排名**）。
- **邊界**：不算特徵、不訓練、**不評分排名**；selection 以資料正確+精準為唯一判準。

### 4 · model（`src/augur/models/`）
- **職責**：訓練模型（樹家族為主軸，transformer 為輔）產 artifacts + registry；自建 `model_registry` 表。
- **輸入→輸出**：核心股 × source-pure 特徵 → 模型檔。
- **守**：① 只吃 gate 通過之完整核心股（已保證 0 缺值，#1）；② 訓練/驗證切分無洩漏（#8）。
- **邊界**：不抓 API、不選股；只載入核心股完整面板（缺值股已被 universe gate 排除，無 imputed 可濾）。

### 5 · validate（`src/augur/evaluation/`）
- **職責**：自包含 walk-forward，產各週期可信度 + 經濟價值。
- **輸入→輸出**：核心股 × 特徵（自己 train→predict）→ 各週期 metrics。
- **守**：② purged 口徑（#8）；③ #12 單一引用源、#13 規則地板、#14 經濟價值、#15 誠實。
- **邊界**：**不讀 model 層 artifacts**（雙軌獨立，自做 train/predict）；metric + panel 窗由單一 helper 算（#12）。

### 橫切 · audit（`src/augur/audit/`）
- **職責**：DB↔API byte-level 對帳（無幻像 attestation）+ 訓練前特徵稽核（**五鏡**）。
- **守**：① #7 對帳；③ #11 五鏡、#15 誠實。
- **邊界**：唯讀稽核，不改資料、不選股。

## 第五部　維運矩陣（從零 12-PHASE）

從零重建一律照此序列；**不可跳 phase、不可改 order**，每 PHASE 過 audit gate 才進下一。

| PHASE | 階段 | 主程式 / 動作 | 產出 | Audit Gate |
|---|---|---|---|---|
| 0 | 環境 | venv + `.env` + PostgreSQL role/db | import smoke OK | imports 全過 |
| 1 | Infra bootstrap | `core` schema bootstrap（infra log DDL） | `pipeline_execution_log` / `data_audit_log` | DDL 無誤 |
| 2 | 名冊 | ingestion `--seed` | `TaiwanStockInfo`（全市場名冊） | 名冊非空 |
| 2b | FRED 前置 | FRED fetch | `fred_series` | series 落地 |
| 3 | 宇宙引導 | universe bootstrap（或 roster fallback） | 過渡名單 | 有標的供 PHASE 4 |
| 4 | 全市場全史 sync | ingestion `--all` | raw 表 generic 自動建 + 全史列 | 供應鏈完整 |
| 5 | **Raw 對帳** | audit reconcile | DB↔API byte-level（#7 無幻像） | value_mismatch=0 ∧ extra=0 |
| 6 | 核心候選（raw 完整）| universe build | raw-完整候選名單 | raw 完整度 PERFECT |
| 7 | Feature Store | features build | feature 面板（source-pure，缺值即缺列） | 面板落地 |
| 8 | 最終核心（feature 完整）| universe `--completeness-gate` | pan-historical 完整核心 snapshot（#1 #10） | 核心股全面板 0 缺值 |
| 9 | 特徵 audit（訓練前） | audit 五鏡（#11） | IC/sign/共線/必要性/SHAP/purged | 特徵分診完成 |
| 10 | 模型訓練 | models train | artifacts + registry | registry 有列 |
| 11 | 模型驗證 | evaluation validators | 各週期 metrics + 經濟價值（#8 #12-15） | 人工對照可信度門檻 |

> **三種建法**（為什麼不是全部由 ingestion 建、PHASE 1 只 bootstrap infra）：
> - **① generic auto-schema → API 原始表**（`TaiwanStockPrice`、`fred_series`…）：型別由 API 回應推導、無白名單（#3）。**兩入口**——FinMind ingester（PHASE 2/4）+ FRED fetcher（PHASE 2b，`fred_series` 走這條、非 FinMind ingester）。
> - **② explicit DDL bootstrap → infra log**（`pipeline_execution_log`、`data_audit_log`）：固定 schema 的系統運維表，**PHASE 1** 最早建（其他階段才有地方記 log）。
> - **③ builder 自建 explicit DDL → 計算型內部表**（`feature_*`、`core_universe_*`、`model_registry`）：系統自己算出來的結構、**無 API 回應可推**，由**各自 builder** 在其 PHASE `CREATE IF NOT EXISTS` 建+寫（feature PHASE 7 / universe 6,8 / model 10），**不集中在 PHASE 1**。
>
> 一句話：**#3「無白名單、全 generic」只管 API dataset；內部表用自己的 explicit DDL——infra log 在 PHASE 1 bootstrap、計算型表由各自 builder 自建。**

**Long-running 治權**（PHASE 4/10/11）：
- ≥5 分鐘 → 每 5 分鐘回報（已完成/elapsed/剩餘/已知 metrics/warning）。
- ≥30 分鐘 / 過夜 → 背景 + 進程 watchdog + sentinel；**確認主機不會睡眠**（雲機/WSL2 須設電源）；須有 resume 後路。
- 任何數字寫入須 trace 回 (a)(b)(c)（#15）。

## 第六部　升版規則

- **新增一條不可違反原則** → 升 major/minor（v1.1.0…），同步 `原則精華`。
- **既有原則微修正** → 改該條文字，**不新增 treaty、不為每次稽核升版**（拒絕舊專案 255-treaty 的累積病）。
- **架構/維運變更** → 改本憲章對應部，記一行於修訂歷程。
- **SSOT 鐵律**：靈魂只住 `核心思想`、法律全文只住 `原則精華`、架構/維運只住本憲章；任一概念**只有一個權威家**（#12）。
- **可讀性目標**：永遠維持「30 分鐘讀懂全系統法律」；篇幅膨脹即是警訊。
