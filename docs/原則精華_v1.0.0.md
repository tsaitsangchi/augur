# Augur — 原則精華 v1.0.0（不可違反的 15 條）

**文件性質**：把舊專案 **255 條 treaty + 77 個迭代子節**去重萃取出的核心法律。
每條格式固定：**WHAT（一句話）｜ WHY（為什麼）｜ ENFORCE（怎麼守）**。
這 15 條 = 「系統架構大憲章 v1.0.0」第四部所索引之法律全文（本檔為 SSOT）。違反任一條 = 治權違規。

> **萃取原則**：憲法只記「**現行不可違反的法律**」，不記「怎麼演變到今天」。
> 演變史 → `docs/archive/`（指向舊 repo）。新增一條法律才升版，不為每次稽核升版。

> **三條基石（★）**：15 條皆不可違反；其中 **#1 零幻像、#8 Anti-Leakage、#15 誠實回報** 為三大命脈——**一條守一個敵人**（①假資料 / ②偷看未來 / ③自我欺騙）。記不住 15 條，先守這 3 條；其餘 12 條是它們的支撐。

## A. 資料紀律（Data Discipline）—— 主防敵人 ① 假資料

### 1. 零幻像 / Source-Pure ★（系統的命）
- **WHAT**：任一特徵值必須是「真實 FinMind/FRED API 值經數學轉換」而得；imputed / zero-fill / hardcoded / 推估 = 幻像。**算不出真實值即不寫入（缺列），不存 fake / zero-fill**（扣靈魂「不存沒有 API 來源的值」）。
- **WHY**：用假資料跑模型沒有意義。
- **ENFORCE**：feature 層算不出 → 缺列；universe 層 **completeness gate** 檢查**全部同版面板**，任一面板任一特徵缺值之股即排除核心；模型只吃完整核心股（無 imputed 可濾）。

### 2. API 即權威
- **WHAT**：表名 / 欄位 / 型別 / 大小寫一律以 API 回應為準（逐字）；下游 schema 查 DB `information_schema`，不另立手維白名單。
- **WHY**：手維 registry 會漂移；API 是唯一事實。
- **ENFORCE**：DB↔API byte-level 對帳；schema helper 由 DB 推導。

### 3. 純通用 Ingestion
- **WHAT**：無 dataset 白名單；通用 auto-schema 自動建任意 API 表。`--all` 動態列舉、無 hardcoded 清單。
- **WHY**：不該存在「只認 N 張表」的 sync 引擎。
- **ENFORCE**：任意 `--dataset X` 即建表；治權排除（intraday / 範圍外）以**排除清單**為之，非正向白名單。

### 4. 日為最小單位
- **WHAT**：不收 tick / 分K / 5 秒；intraday 來源只存日級衍生值（每日一列）。
- **WHY**：本系統以「日」為預測最小單位。
- **ENFORCE**：intraday dataset 守門拒絕入庫。

### 5. 型別紀律
- **WHAT**：字串最小 `VARCHAR(255)`、超過 255 改 `TEXT`（無中間寬度）；數字最小 `NUMERIC(20,6)`、超出自動擴大 precision/scale。
- **WHY**：一致、可預期、不截斷不溢位。
- **ENFORCE**：generic_schema 推導 + 既有表 auto-widen（只擴不縮）。

### 6. 冪等 + 斷點續傳
- **WHAT**：同範圍重跑安全；中斷可從 DB 最後狀態續抓。
- **WHY**：長跑 sync 必須可恢復、不重複、不掉資料。
- **ENFORCE**：`ON CONFLICT DO UPDATE` + DB-driven resume + 主鍵首建固定（不被小樣本推窄）。

### 7. DB↔API 對帳 Attestation
- **WHAT**：定案資料須 byte-level 等同 API。對帳分 4 類——matched / `value_mismatch`（DB≠API）/ `missing_in_db`（API 有 DB 無）/ `extra_in_db`（DB 有 API 無）；attestation 通過 = `value_mismatch=0 ∧ extra_in_db=0`。
- **WHY**：DB 無幻像之可驗證鐵證；API 即權威（#2），DB 應與之一致。
- **ENFORCE（偵測 + 處置）**：reconcile audit 偵測差異後——
  - `value_mismatch` / `missing_in_db`（定案資料）→ **用同一條 sync 路徑重抓該 (股, dataset, 區間)**（強制 re-fetch、繞過 #6 resume 的「已同步即跳過」）→ 冪等 `ON CONFLICT DO UPDATE` 自然覆蓋為當前 API 值。**無獨立修正程式：correction＝重跑正常 sync**（同一 upsert 路徑，守 #1 no-hand-patch、#2 API 權威、#6 冪等）。
  - `extra_in_db`（DB 有 API 無）→ **疑似幻像 / PK 碰撞之紅旗 → 調查根因，不自動補也不自動刪**（亦可能為已下市、API 不再回傳之合法歷史）。
  - 「今日當前交易日（仍結算中）」差異 → 可接受，次日自動刷新。
  - 全部差異**記錄留存**（哪筆 / 舊→新值 / 偵測時點）供 audit，區分 restatement（合法）vs sync bug（須修程式）。

## B. 建模紀律（Modeling Discipline）—— 建模階段橫跨三敵人（#8②、#9①、#10①③、#11 #12③）

### 8. Anti-Leakage
- **WHAT**：只用 as-of ≤t 已知資訊、t+1 套用；walk-forward IC **必附 purged 口徑**（剔除標籤窗與 test 重疊之訓練面板）。
- **WHY**：用了決策當下尚未實現的 label = 樂觀偏誤 = 假成績。
- **ENFORCE**：發布日 gate + embargo purged-CV；**含 survivorship / 選股偏差防護**（歷史宇宙須含當期已下市股，不只「活到今天」的；選股本身不用未來資訊）；單報 raw IC 視為違規。

### 9. 思想 ≠ 特定值
- **WHAT**：可受 Pareto / 康波等思想啟發；但**不把特定值硬編進特徵**——八二法則 `0.80 / 0.20`（含 20/60/20、10% decile 等衍生切點）、康波 `40-60 年`、theme keyword 分數，皆禁入 feature 公式。
- **WHY**：特定預測值 = AI 幻像；思想 ≠ 預測。
- **ENFORCE**：feature 公式禁 hardcoded 知識字典 / 閾值；operational 層（風控配比）之特定值須透明揭露且不入 feature。

### 10. 核心股：質 > 量
- **WHAT**：核心股以資料正確（source-pure）+ 精準（合理性，排離群/錯誤值）為最高判準；**可以變少**。
- **WHY**：要的不是核心股多，是資料對且精準。
- **ENFORCE**：pan-historical 完整度 gate（核心 ＝ 全部 source-pure 完整股，**不評分、不排名上限**）；不以核心股數量為目標。

### 11. 五鏡特徵治理
- **WHAT**：判斷特徵存廢**不得單一指標**（尤不得單看 gain）；五鏡合看——有號 IC+sign 穩定 / 共線群 / leave-one-out 必要性 / ensemble SHAP / purged-CV。
- **WHY**：單一指標會把「共線冗餘」誤判為「真噪音」，或留下模型已忽略的死重。
- **ENFORCE**：五 audit harness 合判；「不顯影（SHAP≈0）且 ablation-safe」必移。

### 12. 單一引用源（SSOT）
- **WHAT**：「用哪些 panel」+「metric 怎麼算」由**單一 helper** 決定；禁各 validator 寫死或 inline。
- **WHY**：跨模型比較必須同基準，否則不可比、不可信。
- **ENFORCE**：全 validator import 共用 panel-date + metric helper。

## C. 風險與治理紀律（Risk & Governance）—— 防敵人 ③ 自我欺騙

### 13. 空頭防護：規則地板優先
- **WHAT**：空頭防護**先建規則型風控地板**（波動目標 × 趨勢，不需預測）；regime 預測只當可選增強層。
- **WHY**：實證——規則地板獨力即大幅降回撤；預測當主角反而較差。
- **ENFORCE**：分層歸因（floor vs floor×regime 增量）；增強層須證 ΔMaxDD<0 ∧ ΔCalmar>0 否則標冗餘。

### 14. 經濟價值判定
- **WHAT**：overlay / 擇時 / 操作週期以 MaxDD / Calmar / Sharpe + 逐空頭子期判定，**非 AUC / 準確率**。
- **WHY**：市場擇時極難；準確率高不等於賺錢或保本。
- **ENFORCE**：walk-forward 經濟價值檢驗；最佳操作週期以「淨報酬 × 信任度 × 穩健度 × 風險」綜合，不裸追最高淨報酬。

### 15. 誠實回報 / 無 Aspirational Inscription
- **WHAT**：每個量化數字可 trace 回 (a) 程式輸出 / (b) DB query / (c) API 回應；**不把「計畫 / 未跑完」當成果**寫入文件；stochastic 指標 ≥3 次取統計；**可重現**（固定 `random_state`、pipeline 確定性 → 半年後重跑結果一致，即靈魂的成功定義）。
- **WHY**：文件失真比沒文件更危險（會驅動錯誤決策）。
- **ENFORCE**：inscription 前 source 檢查清單；結果未跑完不入（待 source-traceable 補入）。

## 升版哲學

- **憲法記現行法律**；歷史進 `docs/archive/`。
- **新增一條不可違反原則** → 升版（v1.1.0…）。
- **既有原則的微修正** → 改該條文字，不新增 treaty、不為每次稽核升版。
- 目標：永遠維持「30 分鐘讀懂全系統法律」的可讀性。
