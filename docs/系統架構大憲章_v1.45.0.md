## 第一部　系統本質

Augur 是一套**只用真實資料、誠實預測台股**的系統：從 FinMind / FRED 抓進可溯源的日級資料 → 算成 source-pure 特徵 → 選出資料乾淨的核心股 → 訓練模型預測未來 H 日的**相對強弱**（橫斷面排序 → top-N）→ 以 walk-forward 誠實驗證並附**可信度**。

> 全文見 `系統核心思想_v1.8.0.md`。一句話：**只信真兆。**

| 面向 | 內容 |
|---|---|
| 標的 | 台灣上市櫃個股 |
| 任務 | 橫斷面相對強弱排序（週/月/季/年多尺度）→ top-N |
| 資料 | FinMind API + FRED；**日為最小單位** |
| 產物 | 分數/機率 + 可信度（walk-forward IC / Eff-t / 勝率）——機率=**橫斷面相對機率**口徑（詳第三部 validate「相對機率誠實判準」v1.40.0）;絕對方向機率唯經預言機軸 GATE(詳第三部 validate「預言機誠實判準」v1.42.0) |
| 邊界 | 不日內、不保證獲利、不擇時躲崩盤、不用未來資訊（anti-leakage）、不下單動錢、不存無源值 |

### 資料本質：PostgreSQL＝唯一系統記錄（single system of record）〔v1.26.0；措辭 patch 2026-07-17 依 RULING-2026-002 主文五〕

本專案**一切資料的唯一系統記錄(system of record)，皆為本地 PostgreSQL**——市場 raw、`feature_values`、universe、模型輸出、知識層（`knowledge_*`／`philosophy_*`）、catalog 元資料、乃至未來權限（`app_user`／群組），全部住 PostgreSQL。

> **釐清（AUD-26 補正；`AUGUR-WM v1.0 §WM.9` 權威三分）**：PG 為系統內**權威 Representation** 之存放處，非 Reality 本身。**Reality 之權威＝API 觀測**（外部世界之忠實記錄）；**系統內權威 Representation＝PG**。原措辭「唯一真相來源」易將 Representation 誤稱為真相（Reality／Truth），與 `AUGUR-MC v1.3 §2.1`（Reality 不以被表徵為前提）、`§P2.E4`（禁 Representation 被視為 Reality）用語紀律牴觸，故正名為「唯一系統記錄」。下游 docstring 之 SSOT 引用隨各檔補正時漸改（patch 級）。

- **外部 API（FinMind／FRED／OpenAlex…）為原始擷取源、非資料家**：一律先落地入 PostgreSQL，下游只讀 PostgreSQL（#9 真實來源之 (b) DB query 為主通道；抓取即入庫、不繞道記憶體／檔案當權威）。
- **衍生索引與傳輸工件非 SSOT、可拋棄**：serving 索引（Milvus 等）、JSON／CSV／md 匯入匯出工件，皆得由 PostgreSQL 全量重建或匯入，永不成第二權威家（承 pgvector＝SSOT 索引鐵則、#29b「repo JSON≠來源 SSOT」、#12）。
- **設定與程式碼不承載資料**：`.env`＝機密設定非資料；code＝邏輯非資料；資料只在 DB。
- **跨機遷移＝`pg_dump`**（#30），非搬檔案。
- **定位**：本條為 #9（真實來源）＋#12（單一 SSOT）＋#29b（資料驅動、來源住 DB）之**系統級明文收斂／總綱**，非新增第 21 條法律。

## 第二部　三個敵人（防線總綱）— 為什麼有這些法律

量化系統最容易死在三件事上。20 條法律**全部**是在防這三個敵人（#17 為取得穩健之 precondition：封鎖則無真資料→一切下游停擺）。

### 敵人 ①　假資料（AI 幻像）
- **長相**：imputed / zero-fill / hardcoded 知識 / 推估 / 系統自己補的值——沒有真實 API 來源。
- **致命後果**：用假資料跑出來的模型沒有意義；錯誤從源頭污染一切下游。
- **防線（原則）**：**#1 零幻像 ★** · #2 API 即權威 · #3 純通用 ingestion · #4 日為最小單位 · #5 型別紀律 · #6 冪等+resume · #7 DB↔API 對帳 · #9 思想≠特定值（hardcoded 值＝注入假資料）· #10 核心股質>量（不完整股排除）· #16 clean-room（防 stock_backend 硬編值經 code 回流）· #17 API 速率公民（主動限速,防 IP 封鎖致無真資料）· #18 抓取依 API 探測（方式+範圍、補多維度抓取,確保抓到 API 實有真資料）。
- **最危險的管線層**：`raw`、`feature`、`universe`。

### 敵人 ②　偷看未來（時間洩漏）
- **長相**：用了決策當下還不知道的資訊（未來函數 / 標籤窗洩漏 / 發布日錯置）。
- **致命後果**：回測很美、實戰崩盤——最隱蔽、最致命的自欺。
- **防線（原則）**：**#8 anti-leakage ★**（≤t 設定、t+1 套用、walk-forward IC 附 purged 口徑；含 survivorship/選股偏差防護）。
- **最危險的管線層**：`feature`（特徵時點）、`model`（訓練/驗證切分）、`validate`。

### 敵人 ③　自我欺騙（灌水回報）
- **長相**：把「打算做的」當成果、拿單次極值當定論、單看 gain 判特徵、以準確率宣稱有效、報喜不報憂、**憑「我以為」代替查證——尤其「我以為不重要／不需要」而跳過 → 漏做 → 系統出錯**。
- **致命後果**：文件失真比沒文件更危險——驅動錯誤決策。
- **防線（原則）**：#10 核心股質>量（不誇大覆蓋）· #11 五鏡特徵治理 · #12 單一引用源 · #13 空頭防護規則地板優先 · #14 經濟價值判定 · **#15 誠實回報 ★** · #16 clean-room（防未重建 code 之自欺回流）· #19 可控風險下逐級逼近最佳奇異點（試錯即進步、重覆驗證再定論，防把單次觀測當結論）· #20 自驅動 × 實證決策（憑「我以為」下決策＝自欺；主動必配實證）。
- **最危險的管線層**：`feature`（特徵存廢）、`validate`（成績宣稱）、報告層。

> ★ = 各敵人的**基石防線**（#1 / #8 / #15，即原則精華「三條基石」）——一條守一個敵人；守住這三條，等於守住三個敵人的核心。

## 第三部　管線分層架構 — 系統怎麼建

資料流：`raw → feature → universe → model → validate`，外加兩個橫切層 `core` / `audit`。
**鐵律：各層職責不越界**——選股不算特徵、特徵不抓 API、驗證不讀訓練產物。

### 橫切 · core（`src/augur/core/`）
- **職責**：路徑/設定、DB 連線、**generic_schema（auto-schema SSOT）**、schema 推導 helper（查 DB information_schema）、infra log 表 DDL（PHASE 1 bootstrap）。
- **守**：① — #2 API 即權威、#3 純通用、#5 型別紀律。
- **邊界**：不抓 API、不選股、不算特徵；只提供地基設施。
- **SSOT**：型別規則（字串 VARCHAR(255)/TEXT、數字 NUMERIC(20,6) 自動擴大）唯一住在這裡。

### 1 · raw（`src/augur/ingestion/`）
- **職責**：通用 ingester 把**任意** FinMind/FRED dataset 自動建表落地（無白名單）。
- **輸入→輸出**：API 回應 → DB 原始表（每值 byte-equal API）。
- **守**：① — #1 #2 #3 #4 #6 #7 #17 #18。
- **邊界**：不算特徵、不選股；intraday 不入庫（只存日級衍生）——**#4 日為資料本質排除準則，其餘日級真實資料皆抓、不以維度/非個股排除；唯資料量規模物理界限者除外（分點/權證/鉅額分點 `OUT_OF_UNIT` 排除，判準全文＝原則精華 #3）**。

### 2 · feature（`src/augur/features/`）
- **職責**：只用 source-pure 原始值，算成特徵面板（panel）；自建 `feature_*` 表（`CREATE IF NOT EXISTS`）。
- **輸入→輸出**：raw 表 →（每股每面板）source-pure 特徵；**算不出真實值的不寫入（缺列，不存 fake / 不存 zero-fill）**。完整度（哪些股缺哪些特徵）交由 universe 層判定。
- **守**：① 不造假特徵、**不存無源值**（#1 #9，扣靈魂「不存沒有 API 來源的值」）；② 特徵時點 anti-leakage（#8）。
- **邊界**：不抓 API、不選股；公式禁 hardcoded 知識/閾值（#9）。
- **特徵發現方法（v1.10.0 入憲承載）**：「**如何找出有用的特徵值**」之方法依據——**三鏡頭 × 紀律漏斗**：同一批 raw 欄位經三種旋轉視角（**第一性原理**＝資訊內容/七軸、**八二法則**＝分布形狀/六軸 cutoff-free 泛函、**康波循環**＝時間結構/六軸 data-driven 相位）生成候選假說，再過**四道漏斗 + 系統級收尾關卡**（紀律閘 #1/#8/#9 → 五鏡 #11 → out-of-sample walk-forward → **提拔關卡**〔v1.13.0：入生產前強制 as-of + 去相關 Eff-t（HAC、禁裸 iid）+ 多 seed 多因子增量；Δ≤0 不提拔〕→ **經濟價值驗證**〔v1.14.0 入憲、系統級 #14：特徵集＋模型須過 purged walk-forward 投組之 CAGR/Sharpe/MaxDD/Calmar 對比基準,風險調整後優於基準才算成功——**IC 撐住 ≠ 可交易,靈魂成功定義是經濟價值非 IC**〕）淘汰;**有用＝驗證活下來、非設計出來**。詳法見 `reports/augur_feature_discovery_methodology_20260626.md` §四（憲章承載框架、方法檔承載詳法，守第六部 SSOT + 30 分鐘可讀）；屬 #1/#8/#9/#11/#14/#15 之操作化收斂、非新增原則。後續一切特徵值產生作法以該檔為依據，偏離（漏過漏斗/硬編特定值/洩漏時點/裸用 iid Eff-t/未過提拔關卡逕入生產/未過經濟驗證即宣稱成功）即治權違規。

### 3 · universe（`src/augur/universe/`）
- **職責**：核心股選拔——核心股 ＝ **全部「source-pure 完整」的股**（無評分、無排名上限）；任一面板任一特徵缺值（不完整）即排除；自建 `core_universe_*` 表。
- **完整度判準之兩項限定〔v1.24.0 承載,稽核決5〕**（生產 build 自 2026-06-30 啟用、此前未入憲；性質＝完整度 gate 之邊界限定、非評分排名）：
  - **(a) 流動性分位地板**：以 `dollar_volume_log_20d` 之**動態橫斷面百分位**（如最近 panel P25）為下界剔極不流動股——**動態百分位非硬編絕對值**（守 #9）、非評分（過線即入、線上不排名）；理由＝極不流動股之特徵值雜訊大、不具可交易性。
  - **(b) conditional 結構性豁免**：金融保險業無月營收申報制度 → 對其豁免月營收完整度要求（`--exempt-revenue-financial`），否則整板塊被 gate 結構性誤排（非資料真缺）。
- **輸入→輸出**：raw 表（完整候選）＋ feature 面板（完整度收緊 ＋ 上述兩限定）→ committed 核心股名單（**完整度關，無評分排名**）。
- **守**：① source-pure gate（#1）、不完整股排除（#10）；③ 質>量（#10，可以少、不以數量為目標、**不排名**）。
- **邊界**：不算特徵、不訓練、**不評分排名**；selection 以資料正確+精準（＋流動性可交易性、結構性豁免）為判準。

### 4 · model（`src/augur/models/`）
- **職責**：訓練模型（樹家族為主軸，transformer 為輔）產 artifacts + registry；自建 `model_registry` 表。
- **輸入→輸出**：核心股 × source-pure 特徵 → 模型檔。
- **守**：① 只吃 gate 通過之完整核心股（已保證 0 缺值，#1）；② 訓練/驗證切分無洩漏（#8）。
- **邊界**：不抓 API、不選股；只載入核心股完整面板（缺值股已被 universe gate 排除，無 imputed 可濾）。

### 5 · validate（`src/augur/evaluation/`）
- **職責**：自包含 walk-forward，產各週期可信度 + 經濟價值。
- **輸入→輸出**：核心股 × 特徵（自己 train→predict）→ 各週期 metrics。
- **守**：② purged 口徑（#8）；③ #12 單一引用源、#13 規則地板、#14 經濟價值、#15 誠實。
- **邊界**：**不讀 model 層 artifacts**（雙軌獨立，自做 train/predict）；metric + panel 窗由單一 helper 算（#12）。
- **相對機率誠實判準〔v1.40.0〕**:系統對個股未來走向之機率輸出,**唯一合法口徑=橫斷面相對機率** P(勝過同儕中位數 | as-of, horizon),以 walk-forward OOS 折之(橫斷面分位,已實現標籤)對樣本作 expanding 校準——**校準器僅得 fit 於「標籤窗已完全實現於評估折 as-of 之前」之折(purge),非裸「前折」**——並隨數字揭露可靠度(Brier 對基線/ECE/可靠度分箱,逐折口徑,禁 iid 顯著性宣稱)。**除預言機軸〔v1.42.0〕預註冊過 GATE 之輸出外,禁止輸出或暗示絕對漲跌機率**(未經 GATE 之「N 天會漲的機率」=假兆)。橫斷面口徑聲明、日曆日↔交易日對映偏差、經濟判死標籤、模型同族近似聲明須與機率數字**硬綁呈現**,不得只出數字;分位方向契約(1=最強,與標籤同向)與校準器 provenance(calibrator_id)入庫可溯源。第一部產物列「分數/機率+可信度」即此口徑。

- **預言機誠實判準(絕對方向機率軸)〔v1.42.0〕**:系統得增設**絕對方向機率**輸出軸(預言機軸),與 v1.40.0 相對機率軸並立、**疊加不拆**:
  - **合法輸出形**:P(個股 H 內絕對報酬>0 | as-of)(H 軌:日曆 30/60/120 天之交易日對映 horizon;個股層+top3/top5 組合層)與日粒度方向機率(D 軌:次日/k 日方向)。兩軌 GATE 獨立。

## 第五部　維運矩陣（從零 12-PHASE）

從零重建一律照此序列；**不可跳 phase、不可改 order**，每 PHASE 過 audit gate 才進下一。

> **三種建法**（為什麼不是全部由 ingestion 建、PHASE 1 只 bootstrap infra）：
> - **① generic auto-schema → API 原始表**（`TaiwanStockPrice`、`fred_series`…）：型別由 API 回應推導、無白名單（#3）。**兩入口**——FinMind ingester（PHASE 2/4）+ FRED fetcher（PHASE 2b，`fred_series` 走這條、非 FinMind ingester）。
> - **② explicit DDL bootstrap → infra log**（`pipeline_execution_log`、`data_audit_log`）：固定 schema 的系統運維表，**PHASE 1** 最早建（其他階段才有地方記 log）。
> - **③ builder 自建 explicit DDL → 計算型內部表**（`feature_*`、`core_universe_*`、`model_registry`）：系統自己算出來的結構、**無 API 回應可推**，由**各自 builder** 在其 PHASE `CREATE IF NOT EXISTS` 建+寫（feature PHASE 7 / universe 6,8 / model 10），**不集中在 PHASE 1**。
>
> 一句話：**#3「無白名單、全 generic」只管 API dataset；內部表用自己的 explicit DDL——infra log 在 PHASE 1 bootstrap、計算型表由各自 builder 自建。**
>
> **入口對映**：PHASE 1/2/2b/4/5 由單一入口 `scripts/full_market_sync.py` 串跑內建（檔頭序列自述）；表列「主程式/動作」為其內部階段、非分立 CLI 旗標。

**Long-running 治權**（PHASE 4/10/11）：
- long-running 回報（≥5 分鐘）／防睡眠 watchdog＋sentinel（≥30 分鐘/過夜）／環境前置／resume 後路之工具層規則 **SSOT＝CLAUDE.md #21-#23**，本處僅引、不複述。
- 任何數字寫入須 trace 回 (a)(b)(c)（#15）。
- **Claude usage 經濟原則 + 配額護欄**：批量/長跑受「最小化 usage」經濟總則與配額護欄約束——**執行層（how）省 usage、理解層（what/why）ultracode 窮盡**；近上限暫停、過重置點 resume-safe 續、不被配額硬切半途。完整規則與判據 **SSOT＝CLAUDE.md #28**（本處僅引；演進見修訂歷程 v1.11.0／v1.12.0／v1.15.0）。

| 2 依賴+模組邊界 | Hexagonal/Clean Ports&Adapters(Cockburn/Martin)· Deep Modules(Ousterhout) | 各層職責不越界＋core 不抓 API(第三部)；finmind/generic_schema/catalog＝deep module |

| 4 ML 系統 | Leakage Prevention· Reproducibility· Feature SSOT | #8 anti-leakage(＝北極星②「決策當下真看得到?」逐字命中)；#15 半年重跑一致；#12 單一 helper |

**核心定位**：三敵人（#1/#8/#15）＝ ML 系統設計（leakage / reproducibility）＋資料工程（contracts / lineage）之**領域化身**；augur 為這些理論之**收斂實例**，非應用對象。

**理論指出、可選補之小缺口（#15 誠實揭露）**：① 正式 ports/adapters 介面（Hexagonal；現分層夠用）· ② data-contract schema 版本化 · ③ as-of / bitemporal 形式化（vintage 表）· ④ train/serve 一致（俟上 serving）。

> 詳細逐項對照 + 業界出處見 `reports/augur_architecture_theory_20260615.md`（憲章承載定位框架、背書詳檔承載逐項，守第六部 SSOT）。
