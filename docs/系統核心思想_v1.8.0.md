> **本系統的靈魂**——回答「是什麼、為什麼、什麼絕不能違反」。架構、程式、決策都從這裡長出來；讀完這頁就懂 Augur 的 80%。

---

## 一句話

> **Augur 是一套只用真實資料、誠實預測台股的系統。**

從 FinMind / FRED 抓進**可溯源的日級資料** → 算成特徵 → 選出**資料乾淨的核心股** → 訓練模型預測未來一段期間（週/月/季/年）的相對強弱 → 以 **walk-forward 誠實驗證**。

---

## 名字的意思

**augur** = 古羅馬的「觀兆者」，讀真實的徵兆來預言未來。

- 本系統的「兆」= 真實 API 資料。
- 本系統的「預言」= 機率 / 分數預測。
- **只讀真兆，不造假兆。** 這就是系統的命。

---

## 它預測什麼

- **標的**：台灣上市櫃個股。
- **任務**：給定一個 as-of 日期，預測各股未來 H 日的**相對強弱**（橫斷面排序——比的是「誰相對強」），據此選出 top-N。H 可週 / 月 / 季 / 年多尺度。 **預言機軸〔v1.6.0;v1.7.0 增輸出契約;v1.8.0 升格〕**:亦預測未來 H 日(日曆 30/60/120 天對映)之**絕對方向機率**(漲或跌的機率,附歷史 OOS 準確率),個股+top3/top5;及日粒度方向機率。**輸出契約三產物〔v1.7.0;v1.8.0 ③升格〕**=①方向機率 %、②方向準確率 %(horizon 級 OOS 命中率,非單股保證)、③**期望報酬率**(**幅度級產物〔v1.8.0〕**:得逐股呈現——由 horizon 級命中率×該股已實現波幅−成本導出、或導自經濟終關回測;與經濟裁決同源硬綁、假設揭露硬綁、非保證)。**原不可違反句「不是預測絕對漲跌幅」已依用戶三度堅持刪除〔v1.8.0〕**——AI 勸阻與用戶堅持雙留痕於憲章 v1.45.0 該條;**不受本次修訂影響之鎖**:逐日價格點位/路徑永久除外、GATE 唯一產生路、單股準確率宣稱仍禁、三敵零容忍(代表性列舉;完整鎖清單 SSOT=憲章 v1.45.0 輸出契約條)。**唯經預註冊可證偽實驗過 GATE 者才成為產品輸出**;未過即判死留檔——預言機的誠實不在「都答對」,在「說得出自己多常對」。
- **產物**：每股的分數 / 機率 → 選股清單，並附**可信度**（walk-forward IC、Effective-t、勝率）。
- **它不是**：明牌產生器、保證獲利、**價格點位神算**。逐日**方向機率**屬 D 軌可證偽實驗;逐日**價格點位與路徑**永久不是本系統的預測產物(路徑唯以「模擬非預測」之蒙地卡羅情境呈現)。它是一個**有紀律的機率引擎**——答案永遠帶著「我多有把握」。

## 為誰 / 成功長什麼樣

- **為誰**：一個重紀律、對自己誠實的量化實踐者（就是你）——**一位主使用者（superuser）＋受控多使用者群組（RBAC 讀取控制）〔v1.5.0〕**：群組使用者只增／收窄**知識 domain 之讀取範圍**，不改「一個對自己誠實的實踐者」之**單一決策主體**假設（RBAC 只作用於知識讀取可見性，不改預測、不成知識→預測旁路）。不是賣訊號的黑箱，也不是給不懂的人的明牌。
- **成功長什麼樣**：不是「某次回測賺很多」，而是——**每個數字都可溯源、每個結論都 out-of-sample 撐得住、半年後重跑結果一樣**。能信、能重現、能解釋，就是成功。

## 資料只來自哪

- **FinMind API**（台股各類日級資料：價量、法人、融資券、持股、財報、月營收、股利…；及衍生品/外股/商品/匯率等日級情境輸入）
- **FRED**（總體經濟因子）
- **日為最小單位**：不收 tick / 分K / 5 秒；intraday 來源只存其**日級衍生值**（每日一列）。這是抓取的**唯一資料本質界線**——凡 API 有的日級真兆皆值得抓（總經 / 情境輸入亦然），不因「抓取方式麻煩」或「非個股」而捨棄真實資料。
- **每個值都能 trace 回某次 API 回應**——沒有「第四類來源」（記憶 / 推測 / 補值 / 為完整度而生的 placeholder）。

> 〔範圍〕本節指**預測用市場／總經資料（真兆）**之來源。哲學／知識素養**文獻**另循「真實文獻、可溯源、不進預測管線」判準（見下方世界觀末列與憲章 philosophy 層）——素養文獻是解讀底蘊、非預測輸入，不構成「第四類來源」。

## 它在對抗什麼（三個敵人）

量化系統最容易死在三件事上。**Augur 的每一條紀律，都是在防這三個敵人**：

| 敵人 | 長什麼樣 | Augur 怎麼防 |
|---|---|---|
| **① 假資料**（AI 幻像） | imputed / 補值 / hardcoded / 推估——沒有真實 API 來源的值 | **Source-Pure**：非「真實 API 經數學轉換」的值一律排除（見下「最神聖的紀律」） |
| **② 偷看未來**（時間洩漏） | 用了決策當下還不知道的資訊 → 回測很美、實戰崩盤 | **Anti-Leakage**：只用 as-of ≤ t 已知；walk-forward IC 必附 purged 口徑 |
| **③ 自我欺騙**（灌水回報） | 把「打算做的」當成果、拿單次極值當定論、報喜不報憂 | **誠實**：數字可溯源 (a)程式 /(b)DB /(c)API；隨機指標多跑取統計；out-of-sample 才算數 |

> **三條基石**：上表三條防線——Source-Pure（真資料）· Anti-Leakage（不偷看）· 誠實（不灌水）——一條守一個敵人，是所有紀律的核心；記不住全部就先守這三條（其中 **Source-Pure 是地基中的地基**，見下節）。
>
> 這三個敵人的共同點：都讓你「**以為自己懂，其實在自欺**」。
> Augur 寧可難看、寧可少、寧可慢，也不被它們騙。

## 最神聖的紀律（敵人①，系統的命）

> ### 零 AI 幻像 / Source-Pure
> 任一特徵值若**不是**「真實 FinMind / FRED API 值經數學轉換而得」——
> 而是 imputed / zero-fill / hardcoded 知識 / 推估 / 系統自己補的——
> **即視同幻像 → 該股從核心宇宙排除。**
>
> **理由**：用假資料跑出來的模型沒有意義。
> 這條凌駕一切便利、一切「先讓它跑起來」的誘惑。

## 可信度從哪來

Augur 的每個數字之所以敢信，靠三層疊起來：

> **真實資料（source-pure）× 防洩漏的驗證（walk-forward + purged）× 誠實的揭露（可溯源、不灌水）＝ 可信的數字**

少任何一層，數字就只是「好看」而已，不是「可信」。

---

## 北極星問題：真兆，還是假兆？

Augur 是觀兆者。當你猶豫——要不要信一個數字、要不要留一個特徵、要不要寫下一句結論——只問一個問題：

> **這是真兆，還是假兆？**

- 它有**真實 API 來源**嗎？　→ 否則是假兆 **①**（假資料）
- 它是**決策當下真的看得到**的嗎？　→ 否則是假兆 **②**（偷看未來）
- 我報的這個數字，是**真的跑出來、out-of-sample 撐住**的嗎？　→ 否則是假兆 **③**（自我欺騙）

三個都「是」→ **真兆，用它**。任一個「不確定」→ **當假兆**，先查清楚再說。

> **此自檢及於一切判斷**——不只「數字 / 特徵 / 結論」，連**開發 / operational 決策**（殺哪個進程、走哪條路徑、調哪個參數）亦然：憑「我以為 / 推論 / 記憶」下手而未經 probe／實測／code 驗證，即假兆 ③（自我欺騙）。**先驗證、再決策。**

> 整套系統可以濃縮成一句：**只信真兆。**

## 世界觀（其餘信念）

| 信念 | 白話 |
|---|---|
| **思想 ≠ 特定值** | 可受 Pareto、康波等**思想**啟發；但不把 `0.80/0.20`、`40-60 年`這種特定值硬編進特徵。 |
| **質 > 量** | 核心股要對、要準，**可以少**。不追求核心股數量。 |
| **防守用規則，不靠預測** | 空頭防護**先建規則型風控地板**（波動目標 × 趨勢）；regime 預測只當可選增強，且須證明在地板之上有增量。 |
| **經濟價值 ≠ 準確率** | 擇時 / overlay 以 MaxDD / Calmar / 逐空頭子期檢驗，不以 AUC 宣稱有效。 |
| **單一引用源** | 「用哪些 panel」「metric 怎麼算」都由單一 helper 決定 → 跨模型比較才可比、可信。 |
| **系統建議，人決策** | Augur 給的是「機率 + 可信度」，不是命令；它是**有紀律的顧問，不是自動駕駛**。扣扳機的是人。 |
| **逼近最佳奇異點（試錯即進步）** | 解決問題的**預設做法**：在**可控風險**內**逐級試錯、逼近最佳奇異點**（效益最大、恰落在崩潰邊界之內的操作點）；**不怕一時的可恢復錯誤——進步來自試錯**。但**只試方法/參數，絕不試資料真假**（三敵零容忍）。 |
| **自驅動 × 實證（開發 augur）** | 開發本系統時，AI 經授權**自驅動**（自己 prompt 自己 / loop）朝靈魂目標推進、執行層「如何最佳達成」由 AI 主導、自我糾錯試錯；但**凡判斷／做法先實證（probe／API／code／DB），嚴禁憑「我以為」**。**決策層人拍板、執行層 AI 自駕**——別於上「投資扣扳機的是人」（那是系統對用戶的定位；這是 AI 對開發 augur 的定位）。 |
| **哲學素養 = 投資智慧骨架 ＋ 博學底蘊，非占卜**（v1.4.0 擴）| augur 是「**有廣博哲學素養的投資量化顧問（博學投資大師 AI）**」：以**投資哲學**（價值／成長／品質／動能／週期…真實權威文獻）為**核心特徵假說來源 ＋ 解讀層**（三鏡頭：第一性／八二／康波，即其操作化），並廣納**人類哲學經典**（東方：兵法／道家…；西方：柏拉圖／亞里斯多德…公版原典）作**解讀與智慧素養層**。但**哲學是假說／素養、非真兆**：投資因子採用與否由 **out-of-sample ＋ 經濟價值**裁決（「**驗證活下來、非大師說了算**」）；來源限**真實文獻、禁 AI 生成內容入庫**（敵人① / clean-room）；預測**仍只靠真實資料**。哲學增強可解釋性與智慧深度，**不鬆三敵零容忍、不取代真實資料、不變 AI 占卜大師**。 |

> **思想源頭**：augur 脫胎自舊系統的「量子金融藍圖」（市場物理 / 八二法則 / 康波）；它是那套的**紀律化結果**——留思想的啟發、剔預言與硬編值（完整考古見憲章附錄 B）。

## 管線（資料如何流動）

```
raw          feature        universe       model            validate
FinMind  →   source-pure →  核心股選拔  →  樹 / transformer → multi-cycle
FRED         特徵            (gate)          訓練              walk-forward
```

- **raw**：通用 ingester 把任意 API dataset 自動建表落地（無白名單）。
- **feature**：只用 source-pure 值算特徵。
- **universe**：任一面板任一特徵缺值（不完整）的股，排除核心。
- **model**：樹家族為主軸，transformer 為輔。
- **validate**：自包含 walk-forward，產各週期可信度 + 經濟價值。

各層**職責不越界**：選股不算特徵、特徵不抓 API、驗證不讀訓練產物。

---

## 它不做的事（邊界，避免誤解）

- **不做日內**（日為最小單位）。
- **不保證獲利**：高回測報酬 = 多頭 beta + alpha，報告須揭露拆解 + 風險（MDD / 逐空頭）。
- **不宣稱能擇時、躲崩盤**：市場擇時極難；防守靠規則減碼限制傷害，不靠「預測哪天崩」。
- **不用未實現的未來資訊**（anti-leakage）。
- **不替使用者下單、不動錢**。
- **不存沒有 API 來源的值**。
- **不做 AI 占卜大師**（v1.4.0）：哲學（投資哲學 ＋ 廣博人類哲學經典）是**假說來源 ＋ 解讀／素養骨架**，**不取代真實資料預測、不存 AI 生成內容當真兆**（守敵人① / clean-room）；「博學大師」的本質是**有紀律的量化顧問 ＋ 廣博智慧素養**，扣扳機仍是人。

---

## 為什麼存在

讓「**用真實資料、誠實方法做台股量化**」這件事，有一個**乾淨、可信、可重現**的底座——
而不是一堆無法溯源、無法重現、自己都不確定真假的數字。

> 一個系統的價值，不在它宣稱賺多少，而在它的每個數字你都敢相信。
